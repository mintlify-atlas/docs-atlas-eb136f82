# enki-tui

Sync terminal UI library over raw `crossterm` (not ratatui). Chat framework via `Handler<M>` trait. Optional `markdown` feature for `termimad`+`syntect` rendering.

<Note>
**Architecture note:** This is NOT ratatui-based. It uses raw crossterm with a custom canvas for maximum control and zero async overhead.
</Note>

## Core Modules

### chat

High-level chat UI framework. Eliminates event loop boilerplate.

#### Handler Trait

```rust
pub trait Handler<M> {
    /// Called for each message received from the backend channel.
    fn on_message(&mut self, msg: M, cx: &mut ChatContext);
    
    /// Called when the user submits text (presses Enter).
    fn on_submit(&mut self, text: String, cx: &mut ChatContext);
    
    /// Called when the user requests an interrupt (Escape or new message while one is in-flight).
    fn on_interrupt(&mut self) {}
    
    /// Called when the user confirms quit (double Ctrl+C).
    fn on_quit(&mut self) {}
    
    /// Called for key events before input processing. Return `true` to consume the key.
    fn on_key(&mut self, _code: KeyCode, _modifiers: KeyModifiers, _cx: &mut ChatContext) -> bool { false }
    
    /// Provide autocomplete matches for a query string.
    fn autocomplete(&self, _query: &str) -> Vec<String> { Vec::new() }
}
```

#### ChatContext

Intent-driven API for updating the UI:

```rust
pub struct ChatContext {
    canvas: Canvas,
    stream: StreamBuffer,
    indicator: Indicator,
    panel: WorkerPanel,
    title_lines: Vec<Line>,
    streaming: bool,
}

impl ChatContext {
    // Activity
    pub fn think(&mut self)
    pub fn tool(&mut self, name: String)
    pub fn clear_activity(&mut self)
    
    // Streaming
    pub fn stream(&mut self, text: &str)
    pub fn finish_markdown(&mut self)
    pub fn finish_stream(&mut self)
    
    // Messages
    pub fn print_user(&mut self, text: &str)
    pub fn print_error(&mut self, text: &str)
    pub fn print_info(&mut self, text: &str)
    pub fn print_lines(&mut self, lines: Vec<Line>)
    
    // Status
    pub fn set_status(&mut self, text: String)
    pub fn set_title(&mut self, lines: Vec<Line>)
    
    // Workers
    pub fn add_worker(&mut self, id: String, title: String, tier: String)
    pub fn update_worker(&mut self, id: &str, activity: String, emoji: char)
    pub fn remove_worker(&mut self, id: &str)
}
```

#### Example

```rust
use enki_tui::chat::{Chat, ChatContext, Handler};
use tokio::sync::mpsc;

enum BackendMsg {
    Chunk(String),
    Done,
    Thinking,
}

struct MyApp {
    tx: mpsc::UnboundedSender<String>,
}

impl Handler<BackendMsg> for MyApp {
    fn on_message(&mut self, msg: BackendMsg, cx: &mut ChatContext) {
        match msg {
            BackendMsg::Chunk(text) => cx.stream(&text),
            BackendMsg::Done => cx.finish_markdown(),
            BackendMsg::Thinking => cx.think(),
        }
    }
    
    fn on_submit(&mut self, text: String, cx: &mut ChatContext) {
        cx.print_user(&text);
        cx.think();
        self.tx.send(text).ok();
    }
    
    fn on_interrupt(&mut self) {
        // Cancel backend work
    }
}

let (tx, mut rx) = mpsc::unbounded_channel();
let app = MyApp { tx };

Chat::new("❯ ").run(app, || rx.try_recv().ok())?;
```

```rust

```

**Streaming pattern:**

```rust
let mut canvas = Canvas::new(80, 24);
let mut stream = StreamBuffer::new();

canvas.begin_streaming();

    stream.push(&chunk);
    canvas.update_stream(&stream);

```

## Full Example: Custom Chat App

```rust
use enki_tui::chat::{Chat, ChatContext, Handler};
use enki_tui::style::{Line, Style};
use enki_tui::{Color, KeyCode, KeyModifiers};
use tokio::sync::mpsc;

enum Msg {
    Response(String),
    Error(String),
    WorkerStarted { id: String, title: String },
    WorkerDone(String),
}

struct App {
    tx: mpsc::UnboundedSender<String>,
}

impl Handler<Msg> for App {
    fn on_message(&mut self, msg: Msg, cx: &mut ChatContext) {
        match msg {
            Msg::Response(text) => {
                cx.stream(&text);
            }
            Msg::Error(err) => {
                cx.print_error(&err);
            }
            Msg::WorkerStarted { id, title } => {
                cx.add_worker(id, title, "standard".into());
            }
            Msg::WorkerDone(id) => {
                cx.remove_worker(&id);
            }
        }
    }
    
    fn on_submit(&mut self, text: String, cx: &mut ChatContext) {
        cx.print_user(&text);
        cx.think();
        self.tx.send(text).ok();
    }
    
    fn on_interrupt(&mut self) {
        eprintln!("User interrupted");
    }
    
    fn on_key(&mut self, code: KeyCode, mods: KeyModifiers, cx: &mut ChatContext) -> bool {
        if mods.contains(KeyModifiers::CONTROL) && code == KeyCode::Char('l') {
            cx.print_info("Logs: ~/.enki/logs/enki.log");
            return true; // consume key
        }
        false
    }
    
    fn autocomplete(&self, query: &str) -> Vec<String> {
        vec!["task".into(), "execution".into(), "merge".into()]
            .into_iter()
            .filter(|s| s.starts_with(query))
            .collect()
    }
}

fn main() -> std::io::Result<()> {
    let (tx, mut rx) = mpsc::unbounded_channel();
    let app = App { tx };
    
    // Backend thread
    std::thread::spawn(move || {
        // Process messages from tx, send responses to rx
    });
    
    Chat::new("❯ ")
        .autocomplete_trigger('@')
        .run(app, || rx.try_recv().ok())
}
```

## Best Practices

1. **Use ChatContext methods:** Don't manipulate canvas directly — use `cx.stream()`, `cx.print_*()`, etc.
2. **Finish streams explicitly:** Call `cx.finish_markdown()` or `cx.finish_stream()` when done
3. **Keep on_message fast:** Offload heavy work to background threads
4. **Non-blocking backend poll:** Use `try_recv()` instead of blocking `recv()`
5. **Handle resize gracefully:** The framework handles this automatically
6. **Use workers panel:** Show multi-agent activity for better UX

## Performance Notes

- **No allocations per frame:** Canvas reuses line buffers
- **Incremental rendering:** Only changed lines are redrawn
- **Zero async overhead:** Pure sync code, no futures
- **Raw crossterm:** Direct terminal control, no widget abstractions

This makes enki-tui suitable for high-frequency updates (streaming text, spinners at 12.5 FPS) without flicker or CPU spikes.
