# enki-core

Pure sync state machine. Zero async, zero tokio. Orchestrator, DAG scheduler, SQLite persistence, CoW copy manager, git merge refinery, roles, hashlines.

<Note>
**Design principle:** All operations are synchronous. The orchestrator processes commands and returns events — no spawning, no async I/O. The runtime layer (CLI) handles async execution.
</Note>

## Core Modules

### orchestrator

The main state machine. Processes commands, updates the DAG, and returns events.

#### Orchestrator

```rust
pub struct Orchestrator {
    scheduler: Scheduler,
    db: Db,
    monitor: MonitorState,
    events_dir: Option<PathBuf>,
    session_id: String,
}

impl Orchestrator {
    pub fn new(db: Db, limits: Limits, session_id: String) -> Self
    pub fn session_id(&self) -> &str
    pub fn set_events_dir(&mut self, path: PathBuf)
    pub fn db(&self) -> &Db
    pub fn scheduler(&self) -> &Scheduler
    
    /// Process a command and return events for the CLI layer.
    pub fn handle(&mut self, cmd: Command) -> Vec<Event>
    
    pub fn set_step_session(&mut self, execution_id: &str, step_id: &str, session_id: String)
    pub fn session_ended(&mut self, session_id: &str)
    pub fn worker_counts(&self) -> (usize, usize, usize, usize)
    pub fn running_steps(&self) -> Vec<(String, String, Id)>
    pub fn reconcile_merges(&mut self) -> Vec<Event>
}
```

**Usage:**

```rust
let db = Db::open(".enki/db.sqlite")?;
let limits = Limits::default();
let mut orch = Orchestrator::new(db, limits, "session-abc123".into());

// CLI sends commands
let events = orch.handle(Command::CreateTask {
    title: "Fix auth bug".into(),
    description: Some("User login fails on Safari".into()),
    tier: Tier::Standard,
});

// CLI executes events
for event in events {
    match event {
        Event::SpawnWorker { task_id, .. } => { /* spawn ACP agent */ }
        Event::QueueMerge(mr) => { /* launch refinery */ }
        _ => {}
    }
}
```

### scheduler

Concurrency manager. Enforces tier limits, dispatches ready tasks, tracks running workers.

```rust
pub struct Scheduler {
    limits: Limits,
    executions: HashMap<String, ExecutionState>,
    running: TierCounts,
}

pub struct Limits {
    pub max_workers: usize,
    pub max_heavy: usize,
    pub max_standard: usize,
    pub max_light: usize,
}

impl Scheduler {
    pub fn new(limits: Limits) -> Self
    pub fn add_execution(&mut self, execution_id: Id, project_id: Id, dag: Dag, step_tasks: HashMap<String, Id>)
    
    /// One scheduling pass. Returns actions for the runtime to execute.
    pub fn tick(&mut self) -> Vec<SchedulerAction>
    
    /// Two-phase completion: worker finishes (frees tier slot) ...
    pub fn step_worker_done(&mut self, execution_id: &str, step_id: &str, output: Option<String>)
    /// ... then merge lands (fires Merged edges)
    pub fn step_completed(&mut self, execution_id: &str, step_id: &str, output: Option<String>)
    
    pub fn step_failed(&mut self, execution_id: &str, step_id: &str)
    pub fn retry_node(&mut self, execution_id: &str, step_id: &str) -> bool
    pub fn pause_execution(&mut self, execution_id: &str) -> bool
    pub fn resume_execution(&mut self, execution_id: &str) -> bool
    pub fn cancel_execution(&mut self, execution_id: &str) -> Vec<(Id, String)>
    pub fn worker_counts(&self) -> (usize, usize, usize, usize)
}

pub enum SchedulerAction {
    SpawnWorker {
        task_id: Id,
        project_id: Id,
        title: String,
        description: String,
        tier: Tier,
        execution_id: Id,
        step_id: String,
        upstream_outputs: Vec<(String, String)>,
        role: Option<String>,
    },
    TaskBlocked { task_id: Id, execution_id: Id, step_id: String },
    ExecutionComplete { execution_id: Id },
    ExecutionFailed { execution_id: Id },
}
```

**Key pattern:** `step_worker_done()` decrements the tier count (worker process exits). Later, `step_completed()` advances the DAG (merge landed). This two-phase design allows dependents with `Completed` edges to start before the merge finishes.

### dag

Directed acyclic graph with node status tracking and edge conditions.

```rust
pub struct Dag {
    nodes: Vec<Node>,
    index: HashMap<String, usize>,
}

pub struct Node {
    pub id: String,
    pub title: String,
    pub description: String,
    pub tier: Option<Tier>,
    pub status: NodeStatus,
    pub checkpoint: bool,
    pub role: Option<String>,
    pub deps: Vec<Edge>,
    pub dependents: Vec<usize>,
}

pub struct Edge {
    pub target: usize,
    pub condition: EdgeCondition,
}

pub enum EdgeCondition {
    Merged,    // Dep's merge must have landed (status == Done)
    Completed, // Dep's worker finished (status == WorkerDone or Done)
    Started,   // Dep just started running (status == Running, WorkerDone, or Done)
}

pub enum NodeStatus {
    Pending,
    Ready,
    Running,
    WorkerDone, // Worker finished, merge pending
    Done,       // Merge landed
    Failed,
    Blocked,
    Paused,
    Cancelled,
}

impl Dag {
    pub fn single(step_id: &str, title: &str, description: &str, tier: Option<Tier>) -> Self
    pub fn from_steps(steps: &[(String, String, String, Option<Tier>, Vec<String>)]) -> Self
    pub fn from_steps_with_edges(steps: &[(String, String, String, Option<Tier>, bool, Option<String>, Vec<(String, EdgeCondition)>)]) -> Self
    pub fn add_steps(&mut self, steps: &[...]) -> Result<(), String>
    
    pub fn get(&self, id: &str) -> Option<&Node>
    pub fn nodes(&self) -> &[Node]
    pub fn ready_nodes(&self) -> Vec<&str>
    pub fn blocked_nodes(&self) -> Vec<&str>
    
    pub fn mark_running(&mut self, id: &str) -> bool
    pub fn mark_worker_done(&mut self, id: &str) -> bool
    pub fn mark_done(&mut self, id: &str) -> bool
    pub fn mark_failed(&mut self, id: &str) -> bool
    
    pub fn pause_node(&mut self, id: &str) -> Option<bool>
    pub fn resume_node(&mut self, id: &str) -> bool
    pub fn retry_node(&mut self, id: &str) -> bool
    pub fn cancel_node(&mut self, id: &str) -> Option<bool>
    
    pub fn is_complete(&self) -> bool
    pub fn has_failures(&self) -> bool
}
```

**Edge conditions example:**

```rust
// Design → Implement (must wait for merge)
Dag::from_steps_with_edges(&[
    ("design".into(), "Design".into(), "...".into(), Some(Tier::Heavy), false, None, vec![]),
    ("implement".into(), "Implement".into(), "...".into(), Some(Tier::Standard), false, None, 
        vec![("design".into(), EdgeCondition::Merged)]),
    // Test can start as soon as design worker finishes (no need to wait for merge)
    ("test".into(), "Test".into(), "...".into(), Some(Tier::Light), false, None, 
        vec![("design".into(), EdgeCondition::Completed)]),
]);
```

## Full Workflow Example

```rust
// 1. CLI initializes orchestrator
let db = Db::open(".enki/db.sqlite")?;
let mut orch = Orchestrator::new(db, Limits::default(), session_id);

// 2. CLI sends CreateTask command
let events = orch.handle(Command::CreateTask {
    title: "Add rate limiting".into(),
    description: Some("Prevent API abuse".into()),
    tier: Tier::Standard,
});

// 3. Orchestrator returns SpawnWorker event
match &events[0] {
    Event::SpawnWorker { task_id, execution_id, step_id, tier, .. } => {
        // CLI spawns ACP agent via enki-acp
        let session_id = mgr.start_session("claude", &["--acp"], copy_path).await?;
        orch.set_step_session(execution_id, step_id, session_id);
        mgr.prompt(&session_id, "Add rate limiting to the API").await?;
    }
    _ => {}
}

// 4. Worker finishes, CLI reports back
let events = orch.handle(Command::WorkerDone(WorkerResult {
    task_id,
    execution_id: Some(exec_id),
    step_id: Some(step_id),
    title: "Add rate limiting".into(),
    branch: "task/rate-limit".into(),
    outcome: WorkerOutcome::Success { output: Some("Added Redis-based rate limiter".into()) },
}));

// 5. Orchestrator queues merge
match &events[1] {
    Event::QueueMerge(mr) => {
        // CLI runs refinery
        let outcome = merge_branch(repo_path, &mr.branch, &mr.base_branch)?;
        orch.handle(Command::MergeDone(MergeResult { mr_id: mr.id.clone(), outcome }));
    }
    _ => {}
}

// 6. Scheduler ticks again, execution completes
let events = orch.handle(Command::DiscoverFromDb);
assert!(events.iter().any(|e| matches!(e, Event::ExecutionComplete { .. })));
```
